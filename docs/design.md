# Design: TerraDiagnosis / 地脉·镜 — 3D 交互式地形沙盘

Generated by /office-hours on 2026-05-23
Branch: main | Status: APPROVED | Mode: Builder

## Problem Statement

用户在黑客松中构建一个 3D 交互式地形沙盘。灵感来自科技馆的 AR 实体沙盘教具，但做成纯数字版——无需实体沙子，浏览器打开即可使用。用户通过推沙塑造地形，系统实时生成等高线。锁定地形后自动诊断气候（季风），动态形成河流湖泊，完成地理渲染，随后用户可分析地形分类、气候带和水文特征。

## What Makes This Cool

核心体验分两阶段：**创作**和**揭示**。

创作阶段：用户像玩沙子一样推拉地形，实时看到等高线变化。手指一动，山就起来了。

揭示阶段：用户点击「锁定地形」后，系统接管——主导风开始吹，云层撞上山脉变成雨，河流从高山流向低地，湖泊在洼地形成。整个过程是动态的、可观看的——就像看自己创造的大陆「活过来」。

最终的「星球档案」展示地形分类、气候带和水文摘要——这是这个星球的地理身份证。

## Design Decisions

| 决策 | 结论 | 理由 |
|------|------|------|
| 地形形态 | **平面** | 球形渲染复杂度高，平面+边缘山脉自然闭合更现实 |
| 目标设备 | **桌面 Chrome 优先** | WebGL 2.0 / touch API 在移动端差异大；桌面保证 demo 稳定 |
| 音效 | **不做** | 静音 demo 在展台环境中反而更专注 |
| 状态管理 | **Zustand** | 比 Jotai 生态更大，与 R3F 集成文档更多 |
| 重置流程 | **「新建星球」按钮** | 编辑模式和锁定后均可用 |

## Constraints

- 黑客松时间限制 (~48h, 1-2 人)
- 纯浏览器端（无后端依赖）
- 混合观众（技术评委 + 普通观众）
- 目标设备：桌面 Chrome/Edge（WebGL 2.0），最低 4GB RAM
- 构建产物 < 3MB gzipped

## 数据契约

### Heightmap
```typescript
type HeightmapData = Float32Array; // 512×512, range [0, 1], transferred via postMessage
```

### 主线程 → Web Worker
```typescript
type SimRequest = {
  type: "SIMULATE";
  heightmap: Float32Array;
  windDirection: number;     // radians, default π/2 (西风)
  seaLevel: number;          // [0, 1], default 0.15
};
```

### Web Worker → 主线程
```typescript
type SimProgress =
  | { phase: "hydrology"; riverMask: Uint8Array; lakeMask: Uint8Array; flowAccum: Float32Array }
  | { phase: "climate";   precipMap: Float32Array; tempMap: Float32Array }
  | { phase: "complete";  analysis: PlanetArchive };

type PlanetArchive = {
  terrain: { mountainPct: number; basinPct: number; plainPct: number; plateauPct: number };
  climate: { zones: Array<{ name: string; pct: number }>; prevailingWind: string };
  hydrology: { riverCount: number; lakeCount: number; watershedArea: number };
};
```

### Zustand Store
```typescript
type AppState = {
  mode: "edit" | "simulating" | "observing";
  heightmap: Float32Array | null;
  brush: { radius: number; strength: number; type: "raise"|"lower"|"flatten"|"smooth" };
  simProgress: SimProgress | null;
  archive: PlanetArchive | null;
};
```

## 笔刷数学

```
Raise:   h(x,y) += strength × exp(-d²/(2σ²))   where σ = r/3
Lower:   h(x,y) -= strength × exp(-d²/(2σ²))
Flatten: h(x,y) = lerp(h(x,y), avgInRadius, strength)
Smooth:  h(x,y) = lerp(h(x,y), gaussianBlur(h, r), strength)
```

## 着色器等高线

```glsl
// Fragment shader — 叠加在 terrain material 上
float elevation = vWorldPosition.y;
float contourInterval = 0.05;
float halfWidth = 0.3;
float contour = abs(fract(elevation / contourInterval + 0.5) - 0.5) / fwidth(elevation);
float line = 1.0 - smoothstep(0.0, halfWidth, contour);
```

## 两阶段工作流

### 阶段 1: 创作模式 (Edit Mode)
- 鼠标推拉地形（Raise / Lower / Flatten / Smooth 笔刷）
- Shader 实时渲染等高线
- 预设按钮（火山岛、山脉链、陨石坑湖、群岛）
- 「锁定地形」+「新建星球」按钮

### 阶段 2: 模拟模式 (Simulation Mode)
- 高度图 transfer 到 Web Worker
- 水文计算 → 气候计算 → 逐层渲染
- 全屏过渡动画 + 步骤文本流
- 完成后显示「星球档案」面板
- 地形永久锁定，可旋转/缩放观察；「新建星球」重置

## 星球档案（3 项分析）

1. **地形分类**: 山脉 / 盆地 / 平原 / 高原百分比
2. **气候带**: 热带 / 温带 / 寒带 / 干旱带分布
3. **水文摘要**: 主要河流数量、湖泊数量、流域总面积

## 边缘场景与错误处理

| 场景 | 处理方式 |
|------|---------|
| 完全平坦地形 | D8 无流向 → UI 显示「该星球无天然河流」 |
| 闭合盆地 | 流量累积在最低点 → 标记为湖泊 |
| 边界处理 | 高度图边缘强制为 0（海洋），水流可流出边界 |
| Worker 超时 | 30s 超时 → 降级到仅渲染水文 |
| Worker 崩溃 | try/catch → 提示重试，不锁地形 |
| WebGL 上下文丢失 | 监听 contextlost → 显示恢复提示 → 自动重试重建 |

## 核心技术选型

| 层 | 技术 | 理由 |
|---|---|---|
| 3D 渲染 | Three.js + React Three Fiber | 生态最大，文档最全 |
| 地形数据 | Float32Array 512×512 | GPU 友好，transferable |
| 等高线 | Fragment shader (GLSL) | 单次 draw call |
| 水文 | D8 流向 + 流量累积 | O(n)，学术界标准 |
| 气候 | 主导风向 + 地形抬升降水 | 参数少，结果符合直觉 |
| 状态管理 | Zustand | 轻量，R3F 集成好 |
| 模拟线程 | Web Worker + transferable | 主线程不卡顿 |
| 构建 | Vite | Three.js 生态标配 |

## 渲染管线

| 模式 | Material | 输入 |
|------|----------|------|
| 编辑 | `TerrainMaterial`（基础色 + 等高线 shader） | heightmap texture |
| 模拟中 | 同上 + loading overlay | 同上 |
| 观察 | `BiomeMaterial`（气候带着色） | heightmap + precipMap + tempMap |
| 叠加 | 河流线 (`LineSegments`)、湖泊面 (`Mesh`) | riverMask, lakeMask |

## 性能估算

D8 算法（512×512 = 262K 格点，单线程 JS）：
- 洼地填充：~200ms
- D8 流向：~50ms
- 流量累积：~100ms
- 气候模型：~100ms
- **预计模拟总耗时：< 1 秒**

## 设计决策（来自 review 审查）

- **信息架构**: 4 层 — 3D 画布 > 笔刷工具栏 > 预设按钮 > 锁定按钮
- **UI 布局**: 编辑模式左侧笔刷面板 + 中央 3D 画布；观察模式 3D 画布左侧 + 星球档案右侧面板
- **品牌**: 标题 DM Mono / Geist 字族；强调色琥珀 #e8945a
- **笔刷图标**: 4 个 SVG 象形图标（山/谷/平/波）
- **加载态**: 暗色背景 + TerraDiagnosis 标题淡入 + 3D 网格显现
- **空状态**: 平面中心半透明提示「推这片土地，创造你的世界」
- **模拟中**: 全屏 overlay + 步骤文本流（填充洼地 → 计算河流 → 生成气候）
- **Demo 情感弧线**: 好奇 → 惊喜 → 掌控感 → 期待 → 惊叹 → 满足 → 自豪
